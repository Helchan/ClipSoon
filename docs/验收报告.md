# ClipSoon 0.9.5 验收报告

验收日期：2026-07-22
本机环境：macOS 15.7.7、Apple Silicon arm64、Python 3.12.10、PySide6 6.11.1

## 1. 结论

核心功能、搜索契约、持久化、隐私格式、快捷键状态机和 Qt 界面均已完成本机验证；本轮生成的
macOS Apple Silicon 0.9.5 应用包也已完成版本、架构、签名和真实启动验证。0.9.5 针对 Windows
首次快捷键呼出后不失焦、透明窗口 layered backing-store 绘制失败、像素字体产生负 point size、
剪贴板所有者竞争以及监听线程运行中停止增加了平台隔离修复和自动化回归用例。Windows 代码路径
及发布前 Windows runner 测试已就绪，但本机不是 Windows，因此不能把模拟 Win32 边界测试表述为
Windows 10/11 真机通过；Intel macOS 也未生成 universal2 产物。

## 5. UI 与交互验收

0.9.5 使用刚构建的 `dist/ClipSoon.app` 和隔离数据目录完成真实 macOS 启动与窗口冒烟验收。
字体 QSS 已从像素单位换为等效 point 单位，实际面板、搜索框、列表和版本标签的
`pointSizeF()` 分别为 10、16、10、9，均不再是 `-1`；macOS 继续保留原透明投影。Windows 专属
前台句柄、鼠标边沿和 layered-window 兼容路径只能由自动化替身验证，仍需 Windows 真机复验。

0.9.4 使用当时构建的 `dist/ClipSoon.app` 和隔离数据目录完成真实 macOS 界面验收。设置窗口新增
“开机时自动启动 ClipSoon”复选框，三分区、全部字段、数据按钮和保存/取消按钮均在窗口内完整
可见，没有因新增配置产生裁切；主面板右下角正确显示 `v0.9.4`。

主窗口拖动使用非文本空白区域真实操作：初始隔离设置坐标为 `(0, 25)`，拖动 100×100 逻辑像素
并松开后 JSON 自动保存为 `(100, 125)`；关闭整个进程并以相同数据目录重启后，再拖动 20×20
逻辑像素得到 `(120, 145)`，证明不是仅在当前进程内保留位置。自动化用例另外确认搜索框等交互
控件不启动窗口拖动，并将不可见旧坐标约束回当前屏幕可用区域。

开机自启动没有修改验收主机的真实登录项：macOS 测试在临时 home 下验证生成的
`com.clipsoon.app.plist` 包含 `RunAtLoad=true` 和准确 `ProgramArguments`，关闭时删除；Windows
使用模拟 `winreg` 验证 `HKCU\Software\Microsoft\Windows\CurrentVersion\Run` 的 `ClipSoon` 值，
并确认源码优先 `pythonw.exe -m clipsoon`、冻结包只记录应用可执行文件。真正注销/登录后的系统
自动拉起仍保留为两平台真机外部验收项。

真实 macOS 界面已检查 Spotlight 式居中浮层、搜索自动聚焦、文本/截图/文件筛选、图片缩略图
和大图预览、单个文本文件的限量只读预览、完整文件路径、深浅主题设置、`Esc` 隐藏、精确匹配
置顶及乱序查询排除。文件图标在真实滚动绘制中发现并修复了 `QFileIconProvider` 参数类型问题，
修复后文件预览正常；README.md 实际验证只展示前 220 个字符，`...` 在预览框内可见，向下滚动
后内容位置不变，且不显示滚动条或打开编辑器。

右键菜单的紧凑宽度与鼠标悬停态通过 Qt 实际显示和像素采样验证：光标进入菜单项后，该项成为
活动动作并绘制弱于列表蓝色选中态、但与菜单底色有清晰对比的灰蓝背景。

0.9.3 全面复核了应用内会弹出的组件。设置窗口的“呼出方式”和“外观”两个 `QComboBox` 下拉
视图现在直接使用当前 ClipSoon 主题的调色板与 QSS：深色普通项为 `#242630` 背景配
`#F7F7FA` 文字，选中项为 `#5B6CFF` 配白字；浅色普通项为白底深色文字。自动化测试实际展开
下拉视图并抓取像素，确认深色普通行背景正确，不再受浅色 macOS 系统背景影响。历史右键菜单
同步改为接收应用主题，深色菜单、悬停、禁用项及分隔线都有显式颜色；系统托盘菜单继续使用
操作系统原生主题。

底部状态栏已在真实 macOS 主面板验证空闲时完全留空；普通操作和错误反馈由单次计时器显示约
4 秒，后续消息会重置计时，辅助功能授权入口则持续保留到授权完成。

设置窗口已在真实 macOS 界面重排为“快捷键、历史与行为、数据管理”三个紧凑分区：七个输入
控件等宽对齐，数字输入移除突兀的原生步进按钮，三个行为开关压缩为两行；设置窗口不再重复
版本号，主面板右下角显示“↑↓ 选择 | ↵ 发送 | Esc 隐藏 | v0.9.3”。
数据目录入口已在真实 macOS 界面验证：按钮显示“在 Finder 中打开”，点击后设置窗口先关闭，
Finder 成为前台窗口并打开 `~/Library/Application Support/ClipSoon`，不会再被模态设置窗口遮挡。

面板状态策略已通过 Qt 专项验收：默认每次显示都恢复“全部”、清空搜索并选中完整结果第一项；
开启“记住上次状态”后，在默认 3 秒窗口内恢复类型 Tab 和搜索词，再使用条目 ID 恢复完整多选
集合和焦点项；即使搜索无结果也会保留类型与查询。超时或已选条目不存在时恢复“全部”、清空搜索
并回退到第一项；设置控件默认关闭，记忆时长默认 3 秒并限制在 1–300 秒。
真实 macOS 设置界面另外确认了默认关闭时“记忆时长”显示 3 秒且禁用，开启后立即可编辑；
点击“取消”并重新打开设置后仍保持默认关闭，验证了取消操作不会隐式写入配置。0.9.3 的真实
设置窗口已确认开关文字显示为“记住上次状态”。
针对真实隐藏后超时仍恢复旧状态的反馈，记忆过期改为单次计时器主动清理与显示时单调时钟复核的双保险；
后续真实双 Ctrl 路径反馈进一步暴露出，记忆 ID 到期清空后，隐藏中的 Qt 列表控件仍保留旧 `currentIndex`和蓝色选中状态。专项用例先稳定复现出到期后索引仍为第二项，修复后到期回调会在面板仍隐藏时同步清理记忆并重置列表、详情和滚动位置到第一项；若面板已在时限内重新显示，则不会在用户操作期间强制改变选中。
0.9.2 另外使用隔离历史执行真实 macOS 窗口操作：鼠标选择“文本”Tab、输入“状态验收”、方向键
移动到第二项并按 Esc 隐藏。配套的真实 Qt 事件循环验收在 500 ms 后重显时恢复类型、搜索和
第二项焦点；再次隐藏并等待真实 3 秒后，单次计时器主动恢复“全部”、清空搜索并重置到第一项。
双 Ctrl、失焦、Esc 和发送隐藏继续统一在真正 `hide()` 前记录快照与起算时刻，并保留
`hideEvent` 兜底。

文件列表卡顿复现使用 `EverMate_加解密方案.jpg`：文件约 26 MB，尺寸
18068×25534，原同步路径生成 72px 缩略图约 302.1 ms、560×520 预览约 290.4 ms。改为后台解码
后，同一真实文件从 README.md 切换到大图的同步选中耗时为 0.164 ms，后台预览约 296.1 ms 后
无缝出现；再次选中使用缓存，耗时 0.039 ms 且图片立即可见。另以强制休眠 200 ms 的解码器
验证列表缩略图和右侧预览调用均在 20 ms 内返回，完整解码不再位于 GUI 选中路径。

0.9.1 将图片缓存改为按字节预算的 LRU：缩略图 32 MB 且只缓存最终 QPixmap，大图 64 MB、
最多 12 项，窗口预览尺寸按 64 px 分桶；文件修改或大小变化会产生新键，删除历史会清除对应
缓存，快速切换会撤销排队旧任务或丢弃已运行旧结果。以同一张 26 MB、18068×25534 JPG 实测，
同步选中 0.188 ms、后台预览 310.1 ms、缓存复选 0.329 ms，实际大图缓存为 1 项、0.71 MB。
真实 macOS 快速连续下移四项后，选中背景和元信息先更新，最终图片预览正常显示。

最终源码另外通过自动化验证设置页显示物理 Control 语义、Windows Meta round-trip，以及
“取消 / 保存”本地化。macOS 辅助功能权限未由验收过程擅自开启，因此全局双 Ctrl 和跨应用
自动粘贴只验证了状态机、目标恢复与降级逻辑，未宣称完成有权限条件下的端到端验收。
