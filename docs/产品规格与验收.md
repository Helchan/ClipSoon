# ClipSoon 产品规格与验收

## 1. 产品目标

ClipSoon 是一个本地优先、调用迅速的历史剪贴板。用户应能在不离开当前工作流的情况下，
呼出、搜索、预览并发送过去复制过的内容。主窗口不是常驻工作台，而是一个使用时出现、完成
或失焦后隐藏的临时面板。

## 2. 核心用户流程

1. ClipSoon 随应用启动后隐藏并驻留系统托盘；用户开启“开机时自动启动”后，登录系统时自动完成此过程。
2. 用户在任意应用复制文本、图片或本地文件，ClipSoon 在后台记录。
3. 用户双击 `Ctrl`（默认 420 ms 内完成两次短按）呼出面板。
4. 面板出现前记住当前前台应用/窗口；搜索框立即获得焦点。
5. 用户输入关键词，结果按匹配得分排序；同分时按最近复制时间排序。
6. 用户用方向键选择；图片在详情区可预览。
7. 按 `Enter` 或双击结果：ClipSoon 写回剪贴板、隐藏、恢复原目标并发送系统粘贴键。
8. 按 `Esc`、点击外部或切换应用时，面板隐藏但后台继续运行。

## 3. 功能边界

### 必须支持

- macOS 13+、Windows 10/11，Python 3.12。
- 文本、单/多文件、本地图片三类历史。
- 内容哈希去重；相同内容再次复制时更新来源、时间并排到顶部。
- Unicode 不区分大小写搜索；搜索词全部字符必须按顺序出现在条目中。
- 空查询严格按最近复制时间倒序；有查询时严格按匹配得分倒序，同分才按最近复制排序。
- 完全匹配得分最高；其余匹配按查询内容占条目长度比例、连续相邻比例和匹配路径跨度计分。
- 历史列表采用紧凑的 44 px 单行条目，左侧缩略图容器为 30 × 30 px；内容在行内严格垂直居中且上下留白等高，文件图标在缩略图容器内水平和垂直居中且不得越界，不重复显示标题和路径，图片缩略图保持直角矩形、不做圆角裁剪；鼠标所在行以及右键菜单光标所在项均显示弱于蓝色选中态但与底色有清晰对比的半选中背景；支持与 Finder/资源管理器一致的 Shift/Ctrl 鼠标及键盘扩展多选；右键可删除所选或确认后清空全部历史。
- 列表项不显示系统 Tooltip，长文本在列表中单行省略，完整内容只在右侧预览区展示，避免全文浮层跨越屏幕。字体采用接近系统命令面板的层级：搜索框 16 pt、分类 Tab 13 pt、列表主文字、右侧正文预览以及信息区标题/标签/值均为 13 pt；列表使用中等字重而不是小号粗体，信息区标题保留更高字重；列表与预览必须明显可读且辅助信息不得争夺主内容层级。
- 类型筛选既支持鼠标点击，也支持 Tab 按“全部、文本、截图、文件”正向循环和 Shift+Tab 反向循环，切换后搜索框保持可输入。
- 面板每次显示时默认切到“全部”、清空搜索并选中完整结果顶部第一项；“记住上次状态”开关默认关闭，开启后可在 1–300 秒内恢复上次类型 Tab 和搜索内容，并按稳定条目 ID 恢复筛选结果中的完整多选集合和当前焦点项，默认记忆 3 秒；即使搜索无结果也必须恢复类型与搜索内容。计时从面板真正隐藏的时刻开始。双 Ctrl 切换隐藏、失焦自动隐藏、Esc 隐藏和发送后隐藏必须统一在调用 `hide()` 之前记录状态快照和隐藏时刻，`hideEvent` 仅作为系统原生隐藏的兜底且不得重复刷新时间戳。到期时必须主动清空记忆、恢复“全部”并清空搜索内容，并在面板仍隐藏时同步将完整列表、详情和滚动位置重置到第一项，再次显示时使用单调时钟复核；超时、条目已删除或关闭开关时回退到“全部”且无搜索条件下的顶部第一项。
- 主面板不显示独立设置按钮，搜索框左右边框分别与下方内容区左右边界对齐；搜索框使用 16 pt 文字和 30 px 搜索图标，文字可见区域到边框顶部及底部的距离均约为文字高度的一半，避免形成过大的上下空带；点击搜索框左侧的放大镜图标打开设置，系统托盘菜单也保留设置入口。
- 设置窗口按“快捷键、历史与行为、数据管理”分区紧凑排列，字段标签与等宽控件对齐，数字输入不显示突兀的原生步进按钮；版本信息不在设置窗口重复显示，主窗口右下角采用“↑↓ 选择 | ↵ 发送 | Esc 隐藏 | v0.9.7”的低对比度提示；数据目录入口按平台显示“在 Finder 中打开”或“在资源管理器中打开”，点击后先关闭设置窗口再打开目录，避免模态窗口遮挡文件管理器，未保存设置不隐式写入。
- 浅色、深色和跟随系统主题必须覆盖主窗口、设置窗口及其非原生弹出层。设置中的所有下拉列表和历史列表右键菜单必须显式使用当前 ClipSoon 主题的背景与前景色，并为普通、悬停、选中和禁用状态提供清晰对比；不得因系统主题与应用主题不同而出现浅色背景白字或深色背景黑字。
- 底部状态栏空闲时不显示“准备就绪”等占位文字；记录、删除、暂停和错误等反馈显示约 4 秒后自动清除，连续消息以最新一条为准；缺少 macOS 辅助功能授权时保留可点击的设置入口，授权完成后清除。
- 右侧详情上方显示文本、图片或文件预览，下方信息区首行显示类型，次行按类型显示文件完整路径、图片文件大小或文本字数；以本地文件形式复制的单个常见图片也在列表及详情区显示内容预览，同时保留文件发送语义；单个文本文件最多读取开头 4 KB、展示前 220 个字符，未完整展示时末尾另起一行显示 `...`，预览区不响应滚轮、不打开编辑器、不加载完整大文件。
- 图片及图片文件的列表缩略图和右侧大图必须在后台线程解码；点击或键盘移动时先立即更新选中背景、类型和路径，预览区显示加载占位，解码完成后再替换图片；不得因图片尺寸、文件大小或跨项距离阻塞选中反馈。列表缩略图采用 32 MB 字节预算的 LRU 且只保留最终 QPixmap，大图采用 64 MB、最多 12 项的 LRU，预览尺寸按 64 px 分桶；快速切换时撤销排队的旧任务或丢弃旧结果，历史删除时同步失效对应缓存，文件大小或修改时间变化后不得继续命中旧内容。
- 默认双击 Ctrl；可配置双击 Ctrl/Shift/Alt/Meta 或常规组合键。
- “开机时自动启动 ClipSoon”默认关闭；启用后使用当前用户级启动项在 macOS 登录或 Windows 登录时启动，无需管理员权限。源码运行记录当前虚拟环境 Python，打包运行记录 ClipSoon 可执行文件；保存失败时不得把设置误报为已开启，应用每次启动时应刷新已开启启动项中的可执行路径。
- 无标题栏主面板支持从非文本、非列表、非按钮等空白区域按住鼠标左键拖动；搜索框、列表、预览文字、按钮和其他可交互内容保持原行为。仅在实际移动并松开鼠标后持久化窗口左上角逻辑坐标，后续呼出不得重新居中；多显示器布局改变或原屏幕移除后，旧坐标必须约束回当前可用屏幕内。
- Windows 从全局快捷键首次呼出时必须请求并校验原生前台窗口；即使 Windows 拒绝前台切换，也必须在面板显示期间通过前台句柄和鼠标按键边沿识别外部点击，且不得影响面板内点击、设置模态框、右键菜单或“失焦隐藏”关闭配置。
- Windows 主面板必须使用不透明顶层 backing store，彻底避开透明分层窗口的 `UpdateLayeredWindowIndirect` 脏区越界失败；macOS 可继续使用透明窗口和 Qt 投影。界面持久字体使用合法 point size，不得向 Windows 样式传入 `-1`。剪贴板变化不得在通知回调中同步读取 MIME，而应短暂延迟、按原生 sequence 合并并有限重试；全局快捷键监听运行中停止后应按退避间隔自动重建。
- Enter / 双击发送；Windows 恢复目标窗口时仅对已最小化窗口调用 `SW_RESTORE`，普通或最大化窗口的尺寸状态必须保持不变；复制失败时保留内容在系统剪贴板并反馈原因。
- 历史数量、保留天数可配置；图片文件随被清理的历史一并删除。
- 所有历史只保存在本机应用数据目录。
- 权限引导按平台区分：macOS 的辅助功能警告与设置页提供直达系统设置的入口；Windows 不显示该权限要求，只提示管理员权限不对等时的自动粘贴限制。

### 首版明确不做

- 云同步、账户、团队共享、OCR、富文本样式保真、跨设备同步。
- 从 Finder/Explorer 之外的虚拟文件提供器提取尚未落盘的文件。
- 绕过操作系统安全边界；高权限目标窗口和 macOS 辅助功能权限仍受系统约束。

## 4. 可执行验收标准

| 编号 | 场景 | 通过标准 |
| --- | --- | --- |
| A01 | 文本采集 | 复制 Unicode/多行文本后 500 ms 内出现一条文本历史，内容一致 |
| A02 | 文件采集 | 复制单个和多个本地文件后，路径、顺序、数量一致 |
| A03 | 图片采集 | 截图或复制图片后保存为 PNG，列表及详情可见且像素尺寸一致 |
| A04 | 去重 | 同一内容复制两次只有一条记录，时间更新并置顶 |
| A05 | 持久化 | 重启应用后历史、置顶状态和设置仍存在 |
| A06 | 搜索 | 精确最高；短条目高于同连续性的长条目；连续命中高于同长度的离散命中；乱序项排除 |
| A07 | 呼出 | 420 ms 内双击 Ctrl 仅呼出一次；长按、Ctrl+其他键、慢速两次不触发 |
| A08 | 自定义 | 修改快捷键后旧快捷键失效，新快捷键无需重启即可工作 |
| A09 | 键盘发送 | 选中结果按 Enter 后窗口隐藏，原应用收到对应文本/图片/文件 |
| A10 | 鼠标发送 | 双击结果行为与 Enter 相同 |
| A11 | 生命周期 | 初始隐藏；Esc 和失焦隐藏；托盘可再次打开和退出 |
| A12 | 清理 | 超过容量或保留期的最旧未置顶记录删除，关联孤立图片同时删除 |
| A13 | 隐私 | 正常使用无网络请求；数据目录和暂停采集入口清晰可见 |
| A14 | 平台异常 | 权限不足或目标恢复失败时不崩溃，内容仍在系统剪贴板并显示提示 |
| A15 | 状态记忆 | 默认每次显示切到“全部”、清空搜索并选中完整列表第一项；开启记忆后 3 秒内恢复类型 Tab、搜索词、多选和焦点（含无结果搜索），超时或条目失效后恢复“全部”、清空搜索并回退到完整列表第一项 |
| A16 | 主题弹出层 | 浅色与深色设置下两个下拉列表及历史右键菜单的普通、悬停、选中、禁用文字均可辨识，应用主题与系统主题不一致时仍不串色 |
| A17 | 开机自启动 | 默认关闭；开启后写入当前用户级启动项，关闭后删除；源码与打包命令分别正确，注册失败时设置保持原值并反馈原因 |
| A18 | 窗口位置 | 空白区域可拖动，交互控件不可误触发拖动；松开后保存坐标，隐藏重显及重启后位置一致，失效坐标回到可见屏幕 |
| A19 | Windows 生命周期与稳定性 | 首次快捷键呼出后点击外部即隐藏；字体、窗口绘制和剪贴板竞争不导致面板失效；快捷键监听停止后自动恢复；发送后普通/最大化目标窗口保持原尺寸，最小化目标可恢复并接收粘贴 |

## 5. 验证边界

核心搜索、持久化、去重、清理和快捷键状态机可在无头自动化测试中完整验证。窗口视觉、
系统剪贴板格式、全局快捷键权限、跨进程自动粘贴和系统登录启动必须分别在 macOS 与 Windows 真机验收；
macOS 上的本机验收不能替代 Windows 验收。

## 6. 性能预算

- 常驻预热后，快捷键回调到面板 `show()` 完成 p95 小于 100 ms。
- 默认 500 条历史的搜索与排序 p95 小于 20 ms；不允许在输入路径读取 SQLite 或图片原图。
- 列表选中和详情元信息同步更新应在单帧预算内完成；图片缩略图与大图解码不计入该同步路径且不得阻塞 GUI 线程。
- 后台剪贴板变化在 500 ms 内入库；大图 PNG 编码和落盘不得阻塞 GUI 线程。
- 隐藏时的平台健康检查只读取 macOS `NSPasteboard.changeCount` 或快捷键监听状态；Windows 前台句柄和鼠标状态只在面板可见时检查，目标空闲 CPU 小于 0.2%。
- 默认容量 500 条，列表使用虚拟化绘制，仅解码屏幕内可见的图片缩略图。
